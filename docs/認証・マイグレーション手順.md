# 認証・マイグレーション手順

## 0. 招待コードの運用

- 招待コードは信頼できる関係者にメッセージで別途共有する想定です。
- マイグレーション `04_add_invite_codes.sql` で初期コード `ULTLA2025`（100回まで）を投入しています。
- 追加・変更は Supabase SQL Editor で `invite_codes` テーブルを直接編集してください。

```sql
-- 新しい招待コードを追加
INSERT INTO public.invite_codes (code, max_uses) VALUES ('YOUR_CODE', 10);

-- 使用状況を確認
SELECT code, used_count, max_uses FROM public.invite_codes;
```

## 1. マイグレーションの実行

Supabase ダッシュボードで以下を**順番に**実行してください。

1. [Supabase Dashboard](https://supabase.com/dashboard) にログイン
2. プロジェクトを選択
3. **SQL Editor** を開く
4. 以下のファイルを順に実行:
   - `supabase/migrations/03_create_profiles.sql`
   - `supabase/migrations/04_add_invite_codes.sql`
   - `supabase/migrations/05_add_caption_to_slides.sql`
   - `supabase/migrations/09_access_control_visibility_invite_codes.sql`
   - `supabase/migrations/10_add_is_admin_to_profiles.sql`

または、Supabase CLI を使っている場合:

```bash
supabase db push
```

## 2. メール認証の設定（任意）

サインアップ時にメール確認を必須にするかどうかは、Supabase の設定で変更できます。

- **Authentication** → **Providers** → **Email**
- 「Confirm email」をオフにすると、登録後すぐにログイン可能になります（開発時におすすめ）

## 3. プレミアムユーザーの設定

特定のユーザーをプレミアムにするには、SQL Editor で以下を実行します。

```sql
-- メールアドレスで指定する場合
UPDATE public.profiles
SET plan = 'premium', updated_at = NOW()
WHERE id = (
  SELECT id FROM auth.users WHERE email = 'your-email@example.com'
);
```

または、ユーザー ID が分かっている場合:

```sql
UPDATE public.profiles
SET plan = 'premium', updated_at = NOW()
WHERE id = 'ユーザーのUUID';
```

## 5. 管理者ユーザーの設定（/admin アクセス用）

管理画面 `/admin` にアクセスできるユーザーは、`public.profiles.is_admin` が `true` のユーザーのみです。

```sql
UPDATE public.profiles
SET is_admin = true, updated_at = NOW()
WHERE id = '管理者にしたいユーザーのUUID';
```


## 4. 動作確認

1. `npm run dev` で開発サーバーを起動
2. トップページ右上の「ログイン」をクリック
3. 会員登録（メール + パスワード）で新規登録
4. ログイン後、スライド詳細で全ページ表示されることを確認
5. プレミアムユーザーに設定したアカウントで「PDF をダウンロード」ボタンが表示されることを確認
