# BASIC / PRO 位置づけとオンボーディングの整理メモ

**作成日**: 2026-02-28  
**更新**: 方針確定 — ログイン＝BASIC、ADVANCE に用語統一、招待コード動線を整理

---

## 方針確定: ログイン＝BASIC、サブスクのみ3段階（ADVANCE に統一）

### 決まったこと

1. **ログインできたら BASIC アカウント**
   - 決済情報の登録は不要 → ハードルが低い。

2. **BASIC モード（無料・ログインのみ＝サブスクに登録したくない人向け）**
   - **招待コードを持っている人**: お金を払わず、コードに紐づくコンテンツを閲覧可能。
   - **ログインだけした人（BASIC のまま）**: **BASIC コンテンツ**（ログインユーザー向けに設定したフリーコンテンツ）がフル閲覧可能。

3. **招待コードの動線**
   - 招待コードを持っていれば **BASIC モード**（ログインはしてるけどサブスクには登録したくない）でも、**コードに紐づいた特定コンテンツ**が見られる。
   - **招待コードの期限が切れたコンテンツ**や**元から招待コードを持っていないコンテンツ**を見るには、**サブスク（PRO または ADVANCE）に登録してね**、という動線にする。

4. **有料コンテンツの見せ方**
   - **個別購入・レンタルは設けない**。サブスクのみ。プランごとに閲覧できるコンテンツレベルを切り替える。

5. **プランは BASIC / PRO / ADVANCE の3段階（ADVANCE に用語統一）**
   - **BASIC**: ログイン＝BASIC（無料）。決済不要。
   - **PRO**: 月額課金。一定のスライドがフル閲覧（PRO レベルまで）。研修スライド・個人情報含むコンテンツは含めない想定。
   - **ADVANCE**: 月額課金・高め。研修スライドや個人情報コンテンツも含め全コンテンツ一括閲覧＋ダウンロード可。

6. **コンテンツの3レベル（DB: content_tier = basic / pro / advance）**
   - **basic**: ログインユーザー向けフリー。BASIC アカウントでフル閲覧。
   - **pro**: 最近のコンテンツ・完成度の高いもの。PRO プランでフル閲覧。
   - **advance**: 研修で使ったスライド、個人情報コンテンツ、ダイジェスト動画など。ADVANCE プランのみ閲覧・DL可。

## 1. 現状の整理（参考）

### いまの違い（実装済み）

| 状態 | 一覧で見えるスライド | 閲覧できる範囲 |
|------|----------------------|----------------|
| **ゲスト（未ログイン）** | visibility=free のみ | 全スライド 4 枚まで |
| **ログイン・サブスク未加入** | visibility=free ＋ 招待コードで紐づいた invite_only | 紐づきスライド＝全ページ、それ以外＝4 枚まで |

- 招待コードを有効にするにはログインが必須。
- 「ログイン＝BASIC」にしたあとも、招待コードは「お金を払わずコード紐づきだけ見る」手段として残す。

## 2. 確定モデル（サブスクのみ・3段階）

### プラン

| プラン | 条件 | 決済 | 見られる範囲 |
|--------|------|------|--------------|
| **BASIC** | ログイン済み | 不要 | BASIC コンテンツ全ページ ＋ 招待コード紐づきコンテンツ全ページ |
| **PRO** | サブスク加入 | 月額（例: 300〜900円程度） | BASIC ＋ PRO レベルコンテンツ（研修・個人情報含むものは除く） |
| **ADVANCE** | サブスク加入 | 月額（高め） | 全コンテンツ一括 ＋ ダウンロード可 |

### 招待コードの動線（ユーザーへの案内）

- **招待コードを持っていれば** → BASIC モード（サブスク未加入）でも、**コードに紐づいたコンテンツ**が閲覧できる。
- **招待コードの期限が切れた**／**元から招待コードを持っていないコンテンツ**を見たい → **PRO または ADVANCE サブスクに登録**してもらう動線にする。マイページの招待コードページ（期限切れ時）や、4枚制限オーバーレイなどで「サブスク登録で続きを見る」を案内。

### コンテンツレベル（スライド側の設定・DB: content_tier）

| レベル | 例 | 閲覧できるプラン |
|--------|-----|------------------|
| **basic** | ログインユーザー向けフリー、昔のスライド・現場でアレンジ推奨 | BASIC 以上 |
| **pro** | 最近のコンテンツ・完成度の高いもの（研修・個人情報含むものは除く） | PRO 以上 |
| **advance** | 研修で使ったスライド、個人情報コンテンツ、ダイジェスト動画など | ADVANCE のみ |

### ゲスト（未ログイン）

- 一覧: visibility=free のスライドのみ。
- 閲覧: 4 枚まで（現状どおり）。ログインすると BASIC アカウントになり、BASIC コンテンツがフルで見られる。

## 3. 実装・用語との対応（Phase 1〜2 との接続）

- **content_tier**: DB は `basic` / `pro` / `advance` のまま。用語は **ADVANCE に統一**（表示・案内文言で「プレミアム」ではなく「ADVANCE」を使う）。
- **profiles.plan**: `free` は「ゲスト」のみとし、**ログイン時は `basic` を付与**（新規サインアップ時も `basic`）。`pro` / `advance`（旧 premium）はサブスク連携で設定。
- **ヘッダー表示**: ゲスト＝「FREE（ゲスト）」、ログイン＝「BASIC」、サブスク＝「PRO」または「ADVANCE」。

## 4. 次のステップ

1. Phase 3 で「ログイン＝plan=basic」にし、新規サインアップ・既存未課金ユーザーを BASIC 扱いにする。
2. コンテンツレベル（basic / pro / advance）に応じたアクセス制御を、現行の `content_tier` と `hasFullAccessToSlide` に反映する。
3. 決済は「PRO サブスク」「ADVANCE サブスク」の2本立てとし、個別購入・レンタルは実装しない。
4. 招待コード期限切れ・コードなしコンテンツを見たい場合の案内を「サブスクに登録してね」に統一（マイページ、4枚制限オーバーレイなど）。

- 「ログインした時点で最低 BASIC」にすると、**ログイン＝決済のハードル**になり、招待コードだけ使いたい人には負荷になる。

## 2. 位置づけのパターン案

### 案A: ログイン＝常に BASIC 扱い（サブスク必須にしない）

- **ログインしたら「BASIC コンテンツ」はフルで見られる**（決済は別途・任意）。
- ゲスト＝4 枚まで。ログイン＝BASIC コンテンツ全ページ＋招待コードで紐づいたスライドも全ページ。
- **メリット**: ログインのメリットがはっきりする。「ログインすると BASIC コンテンツが全部見られる」と説明しやすい。
- **デメリット**: 「BASIC コンテンツ」を「無料で見られる範囲」として広げると、収益化とのバランスを決める必要がある。

※ ここでの「BASIC」は「プラン名」ではなく「見られる範囲」のイメージ。課金は PRO/PREMIUM から、という整理も可能。

---

### 案B: 現状の延長（ログイン＝FREE、招待コードで拡張）

- **ゲスト**: 4 枚まで（free のみ一覧）。
- **ログイン・未課金**: FREE。招待コードで紐づいたスライドは全ページ、それ以外は 4 枚まで。
- **BASIC 課金**: BASIC コンテンツをフル閲覧（ゲストでは見せない「BASIC 専用コンテンツ」を足すことも可）。
- **PRO 課金**: BASIC ＋ PRO コンテンツのレンタル/購入。プレミアム有料は単品 or PREMIUM で一括。

- **メリット**: 「招待コードだけ使いたい人」はログインだけで済み、決済不要。段階を分けやすい。
- **デメリット**: 「ログインしただけでは 4 枚のまま」のスライドが多く、**ログインのメリットが伝わりにくい**場合がある。

---

### 案C: 決済情報を先に入れて PRO お試し → 継続 or BASIC に落とす

- ログイン後、**決済情報（カード）を登録** → **PRO を 1 週間お試し**。
- お試し終了時:
  - **キャンセルしなければ**: PRO として自動継続・引き落とし開始。
  - **お試し中にキャンセル**: BASIC モード（無料）に戻す。カードは登録のままでも課金なし。
- BASIC を「PRO の下」に置き、**ゲストでは見せず BASIC だけ見せるコンテンツ**を用意する。

- **メリット**: お試しで PRO を体験してもらい、解約時も「何も見られなくなる」ではなく BASIC に落ちるので心理的負荷は小さい。BASIC に「ログインだけでは見られないコンテンツ」を集めれば、ログイン＋BASIC の価値が分かる。
- **デメリット**: **最初に決済情報を求める**ため、心理的・物理的ハードルは一番高い。「ちょっとだけ見たい」層には重い可能性がある。

## 3. ユーザー負荷の観点

| 観点 | 負荷が小さい方 |
|------|----------------|
| **物理的**（入力の手間） | 案B（ログインだけなら決済不要） |
| **心理的**（「いきなりお金」への抵抗） | 案B または 案A（ログイン＝BASIC で、課金は PRO から） |
| **「ログインの意味」が伝わる** | 案A または 案C（ログイン＝BASIC または お試し PRO） |
| **収益化と設計の一致** | 案C（最初にカード登録＋お試し） |

- **「ちょっと見たいだけ」** を最優先するなら **案B** が一番負荷が小さい。
- **「ログインしたら最低限これが見られる」をはっきりさせたい**なら **案A**（ログイン＝BASIC コンテンツまで見られる、課金は PRO/PREMIUM から）。
- **コンバージョン（PRO 契約）を前面に出す**なら **案C**（決済情報→PRO お試し→継続 or BASIC）。

## 4. PRO と BASIC の役割の整理（現時点の理解）

- **PRO**（月額 300〜900 円程度）  
  - 基本的な有料コンテンツ（BASIC コンテンツ＋PRO コンテンツのうち「サブスクに含まれるもの」）が視聴可能。  
  - **プレミアム有料コンテンツ**は PRO だけでは見られず、**単品の購入/レンタル** か **PREMIUM プラン加入** のどちらかで視聴。

- **BASIC**  
  - **案A**: ログイン＝BASIC 範囲（無料）。課金は PRO/PREMIUM から。  
  - **案B**: 無料のまま「FREE」の上位。BASIC 専用コンテンツ（ゲスト非表示）を足すことも可。  
  - **案C**: PRO の下の無料 tier。お試しキャンセル後の受け皿。ゲスト非表示・BASIC のみ表示のコンテンツを置く。

- **ゲストでは見せず BASIC だけで見せるコンテンツ**  
  - 案B・案C で有効。「ログインすると、ここが解禁される」が分かりやすくなる。

## 5. 次のステップ（提案）

1. **方針を一つに絞る**: 上記のどれを「主軸」にするか（A/B/C のどれか、または A+B の折衷など）を決める。
2. **文言をそろえる**:  
   - ログイン後の状態を「FREE」「BASIC」のどちらで呼ぶか。  
   - ヘッダーの「FREE（ゲスト）」と「ログイン後の FREE/BASIC」の説明を一致させる。
3. **招待コードの扱い**:  
   - 「招待コードだけ使いたい人」を残すか（案B 寄り）、  
   - それとも「ログイン＝最低 BASIC（またはお試し PRO）」に寄せて招待コードを補助に回すか。
4. **Phase 3 実装**:  
   - 選んだ案に合わせて、  
     - ログイン時のデフォルト `plan`（`free` のままか、`basic` にするか）、  
     - チェックアウト／お試し期間の有無、  
     - BASIC 専用コンテンツの有無  
   を仕様に落とす。

このメモをベースに、「ログイン＝最低 BASIC か」「決済をいつ求めるか」を決めてもらえれば、Phase 3 の実装内容をそれに合わせて具体化できます。
