# プランのゾーニングと棲み分け（まとめ）

**参照元**: [BASIC_PRO_設計メモ.md](./BASIC_PRO_設計メモ.md)、`lib/access.ts`  
**更新**: 2026-02 時点の確定方針

---

## 1. ユーザー側の「プラン」4段階

| プラン | 状態 | 決済 | ヘッダー表示例 |
|--------|------|------|----------------|
| **FREE（ゲスト）** | 未ログイン | なし | FREE（ゲスト） |
| **BASIC** | ログイン済み・サブスク未加入 | 不要 | BASIC |
| **PRO** | サブスク加入（月額） | 月額 300〜900 円程度想定 | PRO |
| **ADVANCE** | サブスク加入（月額） | 月額高め想定 | ADVANCE |

- **ログインした時点で BASIC**。決済情報は不要。
- 有料は **PRO** と **ADVANCE** の2段階のみ。個別購入・レンタルは設けない。

---

## 2. コンテンツ側の「レベル」（content_tier）

スライドごとに「どのプランまでフル閲覧（4枚超え）を許可するか」を `content_tier` で設定する。

| content_tier | 想定される中身の例 | フル閲覧できるプラン |
|--------------|--------------------|----------------------|
| **basic** | ログインユーザー向けフリー、昔のスライド・現場でアレンジ推奨 | BASIC 以上（＝ログインすれば全ページ可） |
| **pro** | 最近のコンテンツ・完成度の高いもの（研修・個人情報含むものは除く） | PRO 以上 |
| **advance** | 研修で使ったスライド、個人情報コンテンツ、ダイジェスト動画など | ADVANCE のみ |

- **basic** → BASIC ユーザーでも全ページ閲覧可。
- **pro** → PRO または ADVANCE で全ページ閲覧可。BASIC のみのユーザーは 4 枚まで（または招待コード紐づきで全ページ）。
- **advance** → ADVANCE のみ全ページ＋ダウンロード可。PRO 以下は 4 枚まで（または招待コード紐づきで全ページ）。

---

## 3. 一覧に「出る／出ない」（visibility）

スライドの「一覧に表示するか」は **visibility** で制御。**3値**（free / invite_only / **private**）です。

| visibility | 誰に一覧表示されるか |
|------------|----------------------|
| **free** | 全員（ゲスト含む） |
| **invite_only** | 招待コードで紐づいているユーザー、または PRO / ADVANCE ユーザー（有料プランは実質全スライド一覧可） |
| **private** | 管理者のみ（一般ユーザー・ゲストには一覧にも出さない） |

- **private は意図的に残している**。「どの招待コードにも紐づけず、invite_only にしておく」でも運用は可能だが、**「管理者だけが見る下書き・非公開」**を明示するために **private** を DB・管理画面で使い、`canViewSlide` では private のスライドは管理者以外に表示しない。
- ゲストは **visibility=free** のスライドだけ一覧に表示。
- ログイン後は、free ＋ **招待コードで紐づいた invite_only** が一覧に出る。PRO/ADVANCE は invite_only も含めて広く見える想定（実装では `accessibleSlideIds === null` で「有料プラン＝一覧は広く」を表現）。

---

## 4. 「誰が何を全ページ見られるか」の対応表

|  | ゲスト | BASIC | PRO | ADVANCE |
|--|--------|--------|-----|---------|
| **content_tier = basic** | 4枚まで | ✅ 全ページ | ✅ 全ページ | ✅ 全ページ |
| **content_tier = pro** | （一覧に出ない） | 4枚まで or 招待紐づきで全ページ | ✅ 全ページ | ✅ 全ページ |
| **content_tier = advance** | （一覧に出ない） | 4枚まで or 招待紐づきで全ページ | 4枚まで or 招待紐づきで全ページ | ✅ 全ページ＋DL可 |

- **招待コード**: BASIC / PRO ユーザーが、**コードに紐づいたスライド**については、そのスライドの content_tier に関係なく **全ページ閲覧可能**。
- **ADVANCE**: 全コンテンツ全ページ＋ダウンロード可。招待コードに依存しない。

---

## 5. 招待コードの役割（棲み分けの要）

- **「お金を払わず、特定コンテンツだけ見たい」層**  
  → 招待コードを発行し、そのコードを入力したユーザーは **BASIC のまま**、コードに紐づいたスライドだけ全ページ閲覧可能。

- **「招待コードの期限が切れた／そもそもコードがないコンテンツを見たい」層**  
  → **PRO または ADVANCE のサブスクに登録**してもらう動線。  
  マイページの招待コードページ（期限切れ時）や、4枚制限オーバーレイなどで「サブスク登録で続きを見る」を案内。

- **ログインの価値**  
  - ゲスト: 4 枚まで。  
  - ログイン＝BASIC: BASIC コンテンツ全ページ ＋ 招待コード紐づきコンテンツ全ページ。決済なしでここまで見られる。

---

## 6. 課金の棲み分け（シンプル化）

- **課金形態は月額サブスクのみ**。単品購入・レンタルはなし。
- **PRO**: 月額 300〜900 円程度想定。basic ＋ pro コンテンツをフル閲覧。
- **ADVANCE**: 月額高め。全コンテンツ一括＋ダウンロード可。
- 値付け・著作権・運用をシンプルにするため、**プランごとに閲覧レベルを切り替える**形に統一。

---

## 7. 用語・DB との対応

- **表示・案内**: BASIC / PRO / **ADVANCE** に統一（「プレミアム」は使わない）。
- **DB**: `profiles.plan` は `free` | `basic` | `pro` | `advance`（`premium` は advance と同義で運用）。`free` はゲストのみで、ログイン時は `basic` を付与。
- **content_tier**: DB は `basic` / `pro` / `advance` の3値。スライド編集・管理画面で設定。
- **visibility**: DB は `free` | `invite_only` | `private` の3値。private ＝管理者のみ閲覧（一覧にも出さない）。管理画面・API でそのまま使用。

---

このゾーニングに基づき、アクセス制御は `lib/access.ts` の `getAccessContext`・`hasFullAccessToSlide`・`canViewSlide` で実装されている。
