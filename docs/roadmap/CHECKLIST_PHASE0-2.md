# Phase 0〜2 リリース前確認チェックリスト

**対象**: 招待コード期限切れ時のサブスク案内、プラン・content_tier のDB拡張、アクセス制御の拡張  
**目的**: 本番反映またはサブスク実装（Phase 3）に進む前に確認すること

---

## 1. マイグレーション

- [ ] **20_plan_content_tier.sql** が本番 DB に適用済みである
- [ ] `profiles.plan` の CHECK が `free, basic, pro, advance, premium` になっている（Supabase Dashboard の Table Editor で確認可）
- [ ] `slides.content_tier` カラムが存在し、既存スライドは `basic` になっている
- [ ] 既存ユーザーの `profiles.plan` は `free` または `premium` のままである（意図どおりか確認）

---

## 2. 招待コード・マイページ（Phase 0）

- [ ] **有効な招待コードが 0 件のとき**、マイページ「招待コード」ページの下に「プランへの加入」案内（BASIC/PRO/ADVANCE の説明＋「プラン登録へ」ボタン）が表示される
- [ ] 「プラン登録へ」をクリックすると `/mypage/subscription` に遷移し、「準備中」のメッセージが表示される
- [ ] サイドバー（PC・モバイル）に「プラン登録」リンクがあり、同じく `/mypage/subscription` に遷移する
- [ ] **有効な招待コードが 1 件以上あるとき**は、サブスク案内ブロックは表示されない（案内が重複しない）

---

## 3. 管理画面・スライド（Phase 1）

- [ ] 管理画面でスライド一覧に「コンテンツ階層」列があり、BASIC / PRO / ADVANCE が表示される
- [ ] スライド新規作成時に「コンテンツ階層」を選べ、保存後に一覧に反映される
- [ ] スライド編集で「コンテンツ階層」を変更でき、保存後に一覧に反映される
- [ ] 既存スライドは `content_tier` が BASIC になっている（マイグレーションの UPDATE が効いている）

---

## 4. アクセス制御（Phase 2）

### 4.1 現状どおり（plan=free・招待コードあり）

- [ ] ログイン＋**有効な招待コードで紐づいたスライド**は、これまでどおり**全ページ**閲覧できる
- [ ] ログイン＋招待コードなし、または紐づいていないスライドは **4 枚まで**のみ閲覧できる
- [ ] 未ログインでは、visibility=free のスライドが一覧に出て **4 枚まで**閲覧できる

### 4.2 有料プラン（DB で plan を変更しての確認）

- [ ] **plan=advance または plan=premium** のユーザーでログインすると、全スライドが全ページ閲覧可能である
- [ ] 上記ユーザーで「PDF をダウンロード」が表示され、ダウンロード API が 403 にならない
- [ ] **plan=basic** のユーザーでは、`content_tier=basic` のスライドが全ページ閲覧可能である（`content_tier=pro` は 4 枚まで）
- [ ] **plan=pro** のユーザーでは、`content_tier=basic` と `content_tier=pro` のスライドが全ページ閲覧可能である

※ plan の変更は Supabase Dashboard → Table Editor → profiles → 該当ユーザーの `plan` を編集

### 4.3 アカウント表示

- [ ] マイページ「アカウント情報」の「アクセス範囲」が、プラン・招待コードの状態に応じて次のように出るか  
  - 管理者: 「管理者（全スライド・ダウンロード可）」  
  - premium/advance: 「全スライド閲覧・ダウンロード可」  
  - basic/pro: 「BASICプランで閲覧可能」／「PROプランで閲覧可能」  
  - free＋招待コードあり: 「招待コードで閲覧可能」  
  - free＋招待コードなし: 「招待コードを追加するとスライドが表示されます」
