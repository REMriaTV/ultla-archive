# 方法2: 手動変換ガイド

1スライド分を画像ベース表示にする手順です。

---

## Step 1: マイグレーション実行

Supabase ダッシュボードを開き、**SQL Editor** で以下を実行してください。

```sql
-- 画像ベース表示用カラム追加
ALTER TABLE slides
  ADD COLUMN IF NOT EXISTS page_count INTEGER,
  ADD COLUMN IF NOT EXISTS page_image_urls TEXT[] DEFAULT '{}';

COMMENT ON COLUMN slides.page_count IS 'スライドの総ページ数';
COMMENT ON COLUMN slides.page_image_urls IS '各ページの画像URL配列';
```

（または `supabase/migrations/02_add_page_images_to_slides.sql` の内容をそのまま実行）

---

## Step 2: 対象スライドのIDを確認

1. Supabase ダッシュボード → **Table Editor** → `slides`
2. 画像化したいスライドの **id** をメモ（画面の一番左の列）
   - 例: ID 6 = 「GREEN 知られざる顔一 (バナナ)」
   - このあと Storage のフォルダ名と DB 更新で使います

---

## Step 3: PDFを画像に変換

### 方法A: ImageMagick（ターミナル）

```bash
# ImageMagick が入っていない場合（macOS）
brew install imagemagick

# 変換（input.pdf を output フォルダに page_1.png, page_2.png ... で出力）
mkdir output
convert -density 150 input.pdf output/page_%d.png
```

### 方法B: オンラインツール

- [Smallpdf](https://smallpdf.com/pdf-to-jpg) などで PDF → 画像に変換
- 各ページを `page_1.png`, `page_2.png` ... の名前で保存

### 方法C: Adobe Acrobat / プレビュー

- 各ページを画像として書き出し
- 連番で `page_1.png`, `page_2.png` ... にリネーム

---

## Step 4: Supabase Storage にアップロード

### 4-1. バケット作成（初回のみ）

1. Supabase ダッシュボード → **Storage**
2. **New bucket** をクリック
3. 名前: `slides`
4. **Public bucket** にチェック（画像を直接表示するため）
5. **Create bucket**

### 4-2. フォルダと画像をアップロード

**具体例: ID 6 のスライドの場合**

1. バケット `slides` を開く
2. **New folder** をクリック
3. フォルダ名に **`6`**（スライドのID）を入力 → 作成
4. フォルダ `6` を開く
5. もう一度 **New folder** をクリック
6. フォルダ名に **`pages`** を入力 → 作成
7. フォルダ `pages` を開く
8. **Upload file** で、変換した画像をまとめてアップロード  
   （`page_1.png`, `page_2.png` ... の名前で）

**フォルダ構造（ID 6 の場合）:**
```
slides/
  └── 6/
        └── pages/
              ├── page_1.png
              ├── page_2.png
              ├── page_3.png
              └── ...
```

---

## Step 5: 画像の公開URLを取得

1. Storage の `slides` → `6` → `pages` を開く
2. 画像を1つクリック
3. 右側パネルの **URL** をコピー  
   形式: `https://ufjunwelbbzurtilgubt.supabase.co/storage/v1/object/public/slides/6/pages/page_1.png`
4. 全ページ分のURLを、順番通りにメモ帳などに並べておく

---

## Step 6: DBを更新

**方法A: SQL Editor で更新（おすすめ）**

1. Supabase ダッシュボード → **SQL Editor** を開く
2. 以下のSQLを貼り付け、**あなたの環境に合わせて書き換えて**実行

```sql
-- ID 6 のスライドを例に、3ページ分の場合
UPDATE slides
SET 
  page_count = 3,
  page_image_urls = ARRAY[
    'https://ufjunwelbbzurtilgubt.supabase.co/storage/v1/object/public/slides/6/pages/page_1.png',
    'https://ufjunwelbbzurtilgubt.supabase.co/storage/v1/object/public/slides/6/pages/page_2.png',
    'https://ufjunwelbbzurtilgubt.supabase.co/storage/v1/object/public/slides/6/pages/page_3.png'
  ]
WHERE id = 6;
```

**書き換えポイント:**
- `page_count`: 画像の枚数（3ページなら3、12ページなら12）
- `page_image_urls`: 各画像のURLを `'...'` で囲み、カンマ区切りで追加
- `WHERE id = 6`: 対象スライドのID

**方法B: Table Editor で更新**

1. **Table Editor** → `slides` を開く
2. ID 6 の行をクリック
3. 右側の `page_count` に `3`（画像枚数）を入力
4. `page_image_urls` に以下をそのまま貼り付け（URLは実際のものに置き換え）:
   ```
   {"https://ufjunwelbbzurtilgubt.supabase.co/storage/v1/object/public/slides/6/pages/page_1.png","https://ufjunwelbbzurtilgubt.supabase.co/storage/v1/object/public/slides/6/pages/page_2.png","https://ufjunwelbbzurtilgubt.supabase.co/storage/v1/object/public/slides/6/pages/page_3.png"}
   ```

---

## Step 7: 動作確認

1. アプリを起動: `npm run dev`
2. ブラウザで `http://localhost:3000/slide/6` を開く（ID 6 の場合）
3. 画像ビューワーが表示され、最初の4ページはクリア、5ページ目以降はぼかし＋オーバーレイになっていればOK

---

## 自動変換の画質を上げる（オプション）

管理画面の「画像に変換」や新規アップロード時の自動変換で、**GraphicsMagick** と **Ghostscript** をインストールすると、pdf2pic による高画質変換が使われます（縞模様・劣化が軽減されます）。

```bash
# macOS
brew install graphicsmagick ghostscript
```

インストールされていない場合は pdf-to-img にフォールバックします。

## トラブルシューティング

| 現象 | 確認すること |
|------|--------------|
| 画像が表示されない | Storage のバケットが **Public** か確認 |
| 404 | URLのパスが `slides/{id}/pages/page_N.png` になっているか確認 |
| まだPDFが表示される | `page_image_urls` が空でないか、正しい形式か確認 |
| ぼかしが効かない | `page_count` が正しく設定されているか確認 |
