## プロンプト本文

```
Next.js 16 + React + Tailwind CSS で構築したスライドアーカイブサイトで、「シリーズ棚のサムネイルが大きすぎる」問題が解決できません。何度か修正を試みましたが、期待通りに小さく表示されず、セカンドオピニオンをお願いします。

---

### 技術スタック
- Next.js 16 (App Router)
- React 18
- Tailwind CSS v4 (`@import "tailwindcss"`, `@theme inline`)
- Supabase（画像URLは外部Storage）

---

### 問題の詳細

**症状**: 「シリーズ」セクションのサムネイルが、ヒーローセクションのサムネイルと比べて非常に大きく表示される。画面幅に対して約3.5枚しか収まらず、U-NEXT の「つづきを再生」棚のようなコンパクトなサイズにならない。

**希望**: 横に4〜5枚が自然に収まる、コンパクトな棚表示。

**参考**: ヒーローセクションのサムネイルは適切なサイズ感で表示されている。シリーズ棚のサムネイルだけが異常に大きく、デザインバランスを崩している。

---

### 試したこと（いずれも効果なし）

1. **calc による相対指定**
   - `w-[calc((100%-0.75rem*3)/4)]` などで親幅に応じて4〜5枚収まるよう指定
   - 結果: サイズが変わらず大きいまま

2. **固定幅の縮小**
   - `w-[80px]` / `sm:w-[96px]` に変更
   - 結果: 変化なし

3. **固定幅の再調整**
   - `w-[100px]` / `sm:w-[120px]` に変更（現状）
   - 結果: 依然として大きく表示される

4. **親要素の確認**
   - メインコンテナは `max-w-4xl`（約896px）
   - シリーズ棚の親は `flex gap-3 overflow-x-auto px-1 pb-2 snap-x snap-mandatory`

---

### 現在の該当コード

**ファイル**: `components/ProgramShelf.tsx`

```tsx
function SlideThumb({ slide }: { slide: Slide }) {
  const [isHovered, setIsHovered] = useState(false);
  const thumb = slide.image_url ?? slide.page_image_urls?.[0];
  // ... caption の処理 ...

  return (
    <Link
      href={`/slide/${slide.id}`}
      className="group relative flex shrink-0 snap-start flex-col overflow-hidden rounded-lg border border-neutral-200 bg-white shadow-sm transition-all duration-200 hover:-translate-y-1 hover:shadow-lg"
      onMouseEnter={() => setIsHovered(true)}
      onMouseLeave={() => setIsHovered(false)}
    >
      <div className="aspect-video w-[100px] shrink-0 overflow-hidden rounded-md bg-neutral-100 sm:w-[120px]">
        {thumb ? (
          <img
            src={thumb}
            alt={slide.title}
            className="h-full w-full object-cover transition-transform duration-200 group-hover:scale-105"
          />
        ) : (
          <div className="flex h-full w-full items-center justify-center text-neutral-400">
            <span className="text-xs">画像なし</span>
          </div>
        )}
      </div>
      {isHovered && (
        <div className="absolute inset-x-0 bottom-0 bg-gradient-to-t from-black/90 to-transparent p-2 pt-6">
          <p className="line-clamp-1 text-xs font-medium text-white">{slide.title}</p>
          {caption && (
            <p className="mt-0.5 line-clamp-2 text-[10px] leading-tight text-white/80">
              {caption}
            </p>
          )}
        </div>
      )}
    </Link>
  );
}

export function ProgramShelf({ program, slides }: ProgramShelfProps) {
  if (slides.length === 0) return null;

  return (
    <section className="mb-8">
      <h2 className="mb-4 text-lg font-semibold text-neutral-800">{program.name}</h2>
      <div className="flex gap-3 overflow-x-auto px-1 pb-2 snap-x snap-mandatory sm:gap-3">
        {slides.map((slide) => (
          <SlideThumb key={slide.id} slide={slide} />
        ))}
      </div>
    </section>
  );
}
```

**構造の要点**:
- 親: `flex gap-3 overflow-x-auto` の横スクロールコンテナ
- 子（各サムネイル）: `Link` に `flex shrink-0 snap-start flex-col`
- 画像ラッパー: `aspect-video w-[100px] sm:w-[120px] shrink-0`
- 幅指定は内側の `div` にのみあり、`Link` 自体には幅指定なし

---

### 想定される原因（未検証）

- `Link` に幅指定がないため、flex 子として何らかの理由で伸びている可能性
- `aspect-video` と `w-[100px]` の組み合わせで、高さ主導で幅が計算されている可能性
- Tailwind v4 の `@theme inline` やビルド設定で、一部のユーティリティが効いていない可能性
- 親の `flex` や `snap-*` が子のサイズに影響している可能性

---

### 添付資料

3. **コード例**: 可能であれば、修正後の `SlideThumb` または `ProgramShelf` のコード例を提示してください
