# 完全自動化：新規スライド作成の使い方

サイトを開いて PDF をアップロードするだけで、画像変換・保存・登録まで自動で完了します。

---

## 事前準備（初回のみ）

### 1. Supabase Storage バケットを作成

1. Supabase ダッシュボード → **Storage**
2. **New bucket** をクリック
3. 名前: `slides`
4. **Public bucket** にチェック
5. **Create bucket**

### 2. 環境変数を設定

`.env.local` に以下を追加：

```
SUPABASE_SERVICE_ROLE_KEY=あなたのservice_roleキー
```

- Supabase ダッシュボード → **Settings** → **API** → `service_role` のキーをコピー
- ⚠️ このキーはサーバー側でのみ使用し、絶対にクライアントに露出しないこと

### 3. キーワード自動抽出（任意）

`.env.local` に以下を追加すると、PDFの内容からキーワードを自動抽出します：

```
OPENAI_API_KEY=sk-your-openai-api-key
```

- OpenAI API キーが必要（有料・従量課金）
- 未設定の場合は、フォームで入力したキーワードのみ使用

### 4. サーバーを再起動

```bash
npm run dev
```

---

## 使い方

1. ブラウザで `http://localhost:3000/admin` を開く
2. **新規スライド作成** のフォームに入力：
   - **PDFファイル**: アップロードする PDF を選択
   - **タイトル**: スライドのタイトル
   - **プログラム**: ドロップダウンから選択（ULTLA, ROCKET など）
   - **年**: 任意（例: 2015）
   - **キーワードタグ**: カンマ区切りで入力（例: フルーツ, バナナ, ABL）
3. **作成** ボタンをクリック
4. 数秒〜数十秒待つ（ページ数によって変動）
5. 「作成完了」と表示されたら、一覧に新しいスライドが追加されます

---

## 自動で行われる処理

1. スライドレコードを DB に作成
2. PDF を Supabase Storage にアップロード
3. PDF を画像に変換（各ページを PNG に）
4. 画像を Storage にアップロード
5. DB に `page_count` と `page_image_urls` を登録

→ 詳細ページ（`/slide/{id}`）で画像ビューワーがそのまま使えます。
